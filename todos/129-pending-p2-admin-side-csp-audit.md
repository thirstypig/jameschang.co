# Admin-side CSP missing — one-sided cross-repo trust boundary

## Problem Statement
The renderer side (`jameschang.co/bucketlist/`, `jameschang.co/now/`) has a strict CSP:
- `default-src 'self'; connect-src 'self' …; script-src 'self' …; object-src 'none'; base-uri 'self'; form-action 'none'`

The admin side (`thirstypig.com/admin/`) is the side that holds the PAT in sessionStorage and writes to BOTH repos. **No CSP audit has been done on the thirstypig admin context.** If the admin loads a script that egresses, the PAT exfiltrates and the cross-repo trust boundary collapses one-sidedly.

Specific concern: `HitListManager.tsx:33` dynamically loads `https://maps.googleapis.com/maps/api/js?key=...&v=weekly`. The `weekly` channel is a moving target with no SRI, so any compromise of Google's CDN (or a typosquat) injects code into the same origin as the PAT.

**Surfaced by:** architecture-strategist and security-sentinel during /ce:review 2026-05-05.

## Proposed Solutions

### Option A — Audit + tighten admin CSP (recommended)
1. Inspect `thirstypig` admin's current `<meta http-equiv="Content-Security-Policy">` (or absence) at `/admin/index.html`.
2. Define minimum required:
   - `connect-src 'self' https://api.github.com https://maps.googleapis.com https://*.googleapis.com`
   - `script-src 'self' https://maps.googleapis.com 'unsafe-inline'` (TinaCMS likely needs inline scripts; verify)
   - `style-src 'self' 'unsafe-inline'`
   - `img-src 'self' data: https://*.googleapis.com https://*.gstatic.com` (Google Maps tiles + place photos)
   - `frame-ancestors 'none'`
3. Pin Google Maps SDK to a fixed version (drop `v=weekly`):
   ```ts
   script.src = `https://maps.googleapis.com/maps/api/js?key=${GOOGLE_API_KEY}&libraries=places&v=3.62`;
   ```
4. Add SRI hash to the script tag if Google publishes one (they don't always; document the trade-off if not).

**Effort:** Medium (~1.5 hours including testing each manager doesn't break under the tightened CSP)
**Risk:** Medium. CSP can break TinaCMS in surprising ways; test in dev before deploying.

### Option B — Defer
The admin is auth-gated by TinaCloud; only the operator can reach it. The realistic threat (npm supply chain, Google SDK compromise) is unmitigated by Tina auth though. Defer is reasonable but documented risk.
