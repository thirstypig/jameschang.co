AI design tools — Claude Design, Figma AI, Lovable

Three tools at different layers of the stack, all worth a look. Claude Design generates component specs and UI code you wire into your own stack — closest fit for a plain HTML/CSS setup like this one, no React scaffolding. Figma AI (a.k.a. Figma Make / First Draft) lives inside your Figma file and thinks in design primitives — frames, auto-layout, your components — rather than code. Lovable sits at the other end: prompt-to-full-app, scaffolds React + Tailwind + Supabase with routing, auth, and a DB schema.

Running all three on different-shaped problems to feel out where each earns its spot. Watching specifically for: does Figma AI replace the rough-sketch step or just automate the parts I already enjoy? Does Lovable’s 0→1 speed hold up when you need to own and evolve the output afterward? Does Claude Design drift toward a generic AI look or stay crisp across many prompts? Verdicts pending.

Codex alongside Claude Code

Running OpenAI’s Codex CLI agent in a second terminal tab alongside Claude Code, to see whether a multi-agent split makes sense for day-to-day work. The specific hypothesis I keep hearing: Codex is especially strong at test generation. So the experiment is “Claude writes the feature, Codex writes the tests” as a specialization split.

Jury’s out on whether the coordination overhead pays for itself or whether one agent doing both tasks stays cleaner. Early instinct for this repo’s style (small, single contributor, plain static site): one agent deep beats two agents shallow. Want to run it on a harder codebase before I commit to an opinion.
